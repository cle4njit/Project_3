{% extends "base.html" %} {% block content %} What is AAA Testing?

Arrange:

Arrange inputs and targets specified cases. Arrange steps should build the base of the test case. Some questions it asks is if the test requires any objects or special settings, or if it needs to prep a database. Arrange takes care of all of these operations once its executed.


Act:

Act works on the target behavior. Act steps should cover the main section of code to be tested. This could be calling a function or method, or somehow interacting with a web page. Act allows for actions to be more honed in and focused on the target behavior.


Assert: C.

Assert lets the developer know what should be the expected outcomes of the test. Act steps should give the user a concise message of what the code should output. Assert steps verify how correct the output is. Overall, Assertions ultimately determine whether the code is up to spec and passes or fails.
